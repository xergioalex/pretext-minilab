# Voronoi Text Cells

## Overview

Voronoi Text Cells divides a canvas into a Voronoi tessellation generated by 4-6 draggable seed points. Each cell contains text laid out to fit its irregular boundary, with per-line widths computed from the cell's shape at each y-position. Dragging a seed point recomputes the tessellation and reflows all text in real time.

This demo matters because it applies Pretext's per-line variable width to genuinely irregular, dynamically changing shapes. Unlike geometric primitives (circles, hearts), Voronoi cell boundaries are defined by proximity to neighboring points, producing complex polygons that change shape whenever any seed moves.

## Pretext APIs Used

| Function / Type | Purpose |
|---|---|
| `prepareWithSegments(text, font)` | Prepares text with segment data for line-by-line layout. Called once per font size change. |
| `layoutNextLine(prepared, cursor, availableWidth)` | Lays out a single line at the width derived from the cell boundary at that y-position. Called per line per cell. |
| `buildFont(fontSize)` | Constructs the font descriptor for the given pixel size. |
| `LayoutCursor` type | Maintains read position between successive `layoutNextLine` calls within each cell. |

## How It Works

### Step 1: Text Preparation

On mount or when font size changes:

```
const font = buildFont(fontSize);
const prepared = prepareWithSegments(text, font);
```

### Step 2: Voronoi Tessellation

The seed points define a Voronoi diagram. For each pixel (or, more efficiently, for each y-position per cell), the cell boundary is the set of points closer to the cell's seed than to any other seed.

For layout purposes, the cell boundary at each y-position is computed as a horizontal span:

```
function cellWidthAtY(seed, allSeeds, y, canvasBounds) {
  // For this y-row, find the x-range where seed is the closest point
  let xMin = canvasBounds.left;
  let xMax = canvasBounds.right;

  for (const other of allSeeds) {
    if (other === seed) continue;
    // compute the x where equidistance line crosses this y
    // narrow xMin or xMax accordingly
  }

  return { x: xMin, width: xMax - xMin };
}
```

### Step 3: Per-Cell Layout

For each cell, text is laid out line by line within the cell's boundary:

```
for (const seed of seeds) {
  let cursor = { offset: 0 };
  let y = cellTop(seed);
  const cellLines = [];

  while (y < cellBottom(seed)) {
    const { x, width } = cellWidthAtY(seed, seeds, y + lineHeight / 2, bounds);
    if (width < minLineWidth) { y += lineHeight; continue; }
    const line = layoutNextLine(prepared, cursor, width);
    if (!line) break;
    cellLines.push({ line, x, y, width });
    y += lineHeight;
  }
}
```

Each cell has its own cursor, starting from the beginning of the text (or from an offset based on cell index for varied content).

### Step 4: Drag Interaction

When a seed point is dragged, the Voronoi tessellation is recomputed and all cells are relaid out. Pretext's sub-millisecond performance makes this smooth at 60fps.

### Step 5: Auto-Orbit Animation

An optional animation mode orbits seed points around the canvas center, continuously reshaping cells and reflowing text.

## State Management

## Visual Rendering

### Cell Boundaries
Voronoi cell edges are drawn as thin lines using the accent color (`#7c6cf0`) at moderate opacity. The edges form the irregular polygonal borders of each cell.

### Seed Points
Each seed point is rendered as a small filled circle at its position. The active (dragging) seed is highlighted with a larger radius and brighter color.

### Text Lines
Each line of text is positioned at its computed `(x, y)` within its cell. Text is clipped to the cell boundary to prevent overflow. Different cells may use different text excerpts for visual variety.

### Cell Fill
Each cell has a subtle background tint, alternating between shades, to visually distinguish adjacent cells.

## Key Technical Insight

The architectural lesson of Voronoi Text Cells is that **Pretext's per-line variable width makes irregular container shapes possible**. Voronoi cells are convex polygons, but their boundaries change dynamically as seed points move. Computing the text layout for each cell requires knowing the cell's width at every y-position -- a value that depends on the positions of all neighboring seeds.

This is fundamentally impossible with CSS:
- `shape-inside` (unimplemented) only supports predefined shapes
- `shape-outside` only affects float wrapping, not arbitrary container shapes
- No CSS property accepts a per-line width function

Pretext's `layoutNextLine()` does not care where the width comes from. Whether it is a circle equation, a Voronoi boundary computation, or a random number generator, the API is the same: pass a width, get a line. This makes Pretext a layout primitive that can integrate with any geometric system.

## How to Replicate

1. **Prepare text once**: call `prepareWithSegments(text, buildFont(fontSize))` when text or font changes.

2. **Implement Voronoi tessellation**: compute the Voronoi diagram from seed points. For layout purposes, you only need the horizontal span (x-range) of each cell at each y-position.

3. **Compute per-line widths per cell**: for each cell and each y-position, determine the cell's width by finding where equidistance lines with neighboring seeds cross that y-row.

4. **Layout each cell independently**: for each cell, call `layoutNextLine(prepared, cursor, cellWidth)` in a loop. Each cell has its own cursor.

5. **Render cell boundaries**: draw Voronoi edges as lines. Render seed points as circles.

6. **Render text**: position each line at its computed (x, y) within its cell. Clip text to cell boundaries.

7. **Add drag interaction**: on mousedown over a seed, track the seed index. On mousemove, update the seed position and recompute the entire layout. On mouseup, release.

8. **Add auto-orbit**: use `requestAnimationFrame` to move seeds along circular paths. Recompute tessellation and layout each frame.
